# AI惑星戦略箱庭

## 目的

`AI惑星戦略箱庭` は、AIが複数の星間帝国を自律的に運営し、資源採掘・物流・艦隊建造・惑星占領を繰り返すようすを観察する、idle watchable な戦略シミュレーションである。

人間はほとんど観察者であり、ときどき世界に介入して均衡を崩す。

---

## コア体験

このゲームの中心体験は「補給線と艦隊の動きを眺めること」である。

人は主に以下を楽しむ:

- 輸送船が惑星間を行き来する物流の流れ
- 敵が補給線を断ちに来る瞬間
- 工場惑星が生産を止め、帝国が弱体化するようす
- 艦隊同士の衝突と、その余波で支配圏が変わるようす
- 資源の豊富な惑星をめぐる争い
- AIごとの戦略差（攻撃的 / 防衛的 / 物流重視）

---

## 世界モデル

### 惑星

各惑星は以下を持つ:

- `id`, `x`, `z` — 位置
- `resources` — 残存資源量（有限）
- `mineRate` — 採掘速度（惑星ごとに異なる）
- `owner` — 支配帝国（-1 = 中立）
- `structures` — 建設済み構造物リスト

資源は **有限** であり、採掘するほど減少する。ただし惑星は消滅しない。資源が尽きた惑星は採掘不能になるが、占領の対象・前線基地としての価値は残る。

### 惑星間接続

惑星はすべての惑星に到達可能だが、距離に応じて移動時間・燃料コストが変わる。レーンは存在しない（自由航行）。

近傍の惑星間には「頻繁に使われる経路」として視覚的な航路線を表示する（game的にはあくまで表示用）。

## 資源と生産チェーン

```
採掘惑星
  └─ mine() → 資源ストック（惑星上）
       ↓ 輸送船が物理移動（攻撃可能）
     工場惑星
       └─ 資源消費 → 船を生産
            ├─ 輸送船    （資源運搬専用）
            ├─ 攻撃船    （惑星占領・艦隊戦）
            └─ 防衛船    （惑星防衛・護衛）
```

**ポイント:**
- 資源は自動では工場に届かない。輸送船が物理的に運ぶ必要がある
- 輸送船が撃墜されると、資源は宇宙空間に散逸（ロスト）
- 工場は資源がなければ停止する
- 占領した工場惑星の生産はそのまま引き継げる

---

## 構造物

### 採掘施設（Mine）

- 建設コスト: 資源 30
- 効果: `mineRate` を +50% 向上させる
- 上限: 惑星ひとつに 2 基まで

### 工場（Factory）

- 建設コスト: 資源 60
- 効果: 船を生産できる。複数工場で生産速度向上
- 複数の工場が同一惑星にあると、生産キューを並列処理できる

### 防衛砲台（Turret）

- 建設コスト: 資源 40
- 効果: 惑星に接近した敵船に自動攻撃（攻撃船・防衛船の代替）
- 占領されると帝国が引き継ぐ

## 船の種類

初期版は以下の3種類のみ。船設計機能は Phase 3 以降。

### 輸送船（Transport）

| パラメータ | 値 |
|---|---|
| 積載量 | 50 資源 |
| 速度 | 高速 |
| 攻撃力 | なし |
| 防御力 | 低 |
| 生産コスト | 資源 20 |

役割: 採掘惑星から工場惑星へ資源を運ぶ。護衛なしで移動するとリスクが高い。

### 攻撃船（Attacker）

| パラメータ | 値 |
|---|---|
| 積載量 | なし |
| 速度 | 中速 |
| 攻撃力 | 高 |
| 防御力 | 中 |
| 生産コスト | 資源 35 |

役割: 惑星占領、敵輸送船への襲撃、艦隊戦。

### 防衛船（Defender）

| パラメータ | 値 |
|---|---|
| 積載量 | なし |
| 速度 | 低速 |
| 攻撃力 | 中 |
| 防御力 | 高 |
| 生産コスト | 資源 28 |

役割: 輸送船への護衛随伴、重要惑星の防衛、敵艦隊の足止め。

## 戦闘システム

### 惑星占領

- 攻撃船が惑星軌道に到達し、防衛側の兵力（防衛船 + 砲台）を上回ると占領成立
- 占領直後は `strength` が低く、再奪還されやすい（Colony と同じ強度概念）

### 艦隊戦（宇宙空間）

- 移動中の船同士が接近すると自動交戦
- 攻撃力 vs 防御力の単純比較 + ランダム要素
- 輸送船は攻撃能力がないため、護衛なしで戦闘に巻き込まれると一方的に失われる

### 砲台

- 惑星に接近した敵船に自動ダメージ
- 射程内にいる間は毎秒ダメージ

## 帝国の性格

3帝国でスタート。性格は以下の4種からひとつ。

| 性格 | 特徴 |
|---|---|
| `industrialist` | 工場建設優先。序盤は弱いが後半強い |
| `raider` | 敵の輸送船を狙う。補給線破壊が主戦術 |
| `expansionist` | 惑星占領数を最優先。物流が後回しになりがち |
| `fortifier` | 防衛船・砲台を重視。攻めは遅いが落とされない |

---

## AIの判断レイヤー

各帝国AIは以下の3レイヤーで判断する:

### 戦略レイヤー（30秒ごと / Ollama）

- どの惑星を次に狙うか
- どこに工場を建てるか
- 輸送ルートをどう設計するか

### 兵站レイヤー（5秒ごと / ルールベース）

- 輸送船が足りているか
- どの工場に資源を優先して送るか
- 護衛を何隻つけるか

### 戦術レイヤー（毎フレーム / ルールベース）

- 近くの敵船を攻撃するか
- 護衛から離れた輸送船を追跡するか
- 劣勢なら撤退するか

## 可視化

### メインビュー（Three.js 3D）

| 要素 | 表示方法 |
|---|---|
| 惑星 | 球体。サイズ = 資源量。色 = 支配帝国 |
| 資源量 | 惑星周囲のリング幅で表現 |
| 工場 | 惑星上の小さな構造物（BoxGeometry） |
| 砲台 | 惑星上の細い棒状構造物 |
| 輸送船 | 小さな四角い船体（BoxGeometry） |
| 攻撃船 | 細長い三角形（ConeGeometry） |
| 防衛船 | 扁平な八面体（OctahedronGeometry） |
| 航路 | 頻繁に使われるルートにうっすら線 |
| 戦闘 | フラッシュエフェクト |

### 観察HUD

- 各帝国: 惑星数 / 資源 / 船数（種別）/ 現在の意図
- 領土比率バー
- イベントログ（補給線切断、占領、工場停止 etc.）

## 人の介入

| 介入 | 効果 |
|---|---|
| `asteroid_field` | 指定エリアの航行を一時妨害（迂回コスト発生） |
| `resource_vein` | 指定惑星の資源を増加 |
| `emp_pulse` | 指定エリアの船を一時停止 |
| `pirate_wave` | 中立海賊船が輸送船を襲う |

---

## ランダムイベント

- 彗星通過（航路に一時障害）
- 資源ボーナス（採掘量が一時増加）
- 中立帝国の出現（第4勢力が一時的に活動）
- 工場の故障（修理コストが発生）

---

## スコアと勝利条件

- スコア = 支配惑星数 × 時間
- 全帝国の中で最多惑星を N 秒間維持すると「支配的優位」の表示
- 完全勝利: 全惑星を制圧（idle 観察として終わりは明確にしてよい）
- idle 継続モード: 新しいランダム帝国が参入して続行

## 成功条件

- 5分放置しても画面に意味のある動きが続いている
- 輸送船の動きを見て「どこに資源が流れているか」が分かる
- 補給線が切断されて工場が止まる瞬間が見える
- 性格の違う帝国が異なる戦略を取っているのが分かる
- 1回介入すると、しばらく影響が続く
