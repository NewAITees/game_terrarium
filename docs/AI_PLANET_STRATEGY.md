# AI惑星戦略箱庭

## 目的

`AI惑星戦略箱庭` は、AIが複数の星間帝国を自律的に運営し、資源採掘・物流・艦隊建造・惑星占領を繰り返すようすを観察する、idle watchable な戦略シミュレーションである。

人間はほとんど観察者であり、ときどき世界に介入して均衡を崩す。

---

## コア体験

このゲームの中心体験は「補給線と艦隊の動きを眺めること」である。

人は主に以下を楽しむ:

- 輸送船が惑星間を行き来する物流の流れ
- 敵が補給線を断ちに来る瞬間
- 工場惑星が生産を止め、帝国が弱体化するようす
- 艦隊同士の衝突と、その余波で支配圏が変わるようす
- 資源の豊富な惑星をめぐる争い
- AIごとの戦略差（攻撃的 / 防衛的 / 物流重視）

---

## デザイン哲学 — リアリティレベル

このゲームは **リアリティレベルを高く保つ** ことを設計の根幹に置く。

### 基本方針

- **星にあるものはすべて使える** — 資源・施設・防衛設備は星に属し、占領した帝国がそのまま引き継ぐ
- **資源は星のもの** — 資源は帝国の口座ではなく惑星ストックとして物理的に存在する。輸送船が物理的に運ぶことで初めて工場に届く。輸送船が撃墜されれば資源はロストする
- **ゲーム的演出を最小化する** — フローティングスコア・派手なフラッシュ・過剰な強調表示は避ける
- **シネマティックなリアリズム** — 演出の目標は「宇宙ドキュメンタリー」。情報は間接的に伝える（工場の灯り・航路の密度・船の群れ）
- **資源は単一種** — 複数資源による複雑化はしない。すべての建造・生産は同一資源で行う

### 演出の方向性

| 避けるもの | 目指すもの |
|---|---|
| フローティングスコアテキスト | 光の流れで航路密度を表現 |
| 派手な爆発フラッシュ | 小さく実在感のある閃光 |
| 抽象的なゲージUI | 惑星サイズ・リング幅による状態表現 |
| 色で情報を過剰強調 | 間接的な視覚情報（灯り・密度・陰影） |

### 資源帰属ルール

- 資源は惑星ストックとして物理的に存在する
- 輸送船が積んだ資源は、撃墜されればロストする
- 惑星を占領すると、その惑星のストック資源をそのまま奪える
- 施設（工場・タレット・採掘施設）も占領側にそのまま引き継がれる

## 世界モデル

### 惑星

各惑星は以下を持つ:

- `id`, `x`, `z` — 位置
- `resources` — 残存資源量（有限）
- `mineRate` — 採掘速度（惑星ごとに異なる）
- `owner` — 支配帝国（-1 = 中立）
- `structures` — 建設済み構造物リスト

資源は **有限** であり、採掘するほど減少する。ただし惑星は消滅しない。資源が尽きた惑星は採掘不能になるが、占領の対象・前線基地としての価値は残る。

### 惑星間接続

惑星はすべての惑星に到達可能だが、距離に応じて移動時間・燃料コストが変わる。レーンは存在しない（自由航行）。

近傍の惑星間には「頻繁に使われる経路」として視覚的な航路線を表示する（game的にはあくまで表示用）。

## 資源と生産チェーン

```
採掘惑星
  └─ mine() → 資源ストック（惑星上）
       ↓ 輸送船が物理移動（攻撃可能）
     工場惑星
       └─ 資源消費 → 船を生産
            ├─ 輸送船    （資源運搬専用）
            ├─ 攻撃船    （惑星占領・艦隊戦）
            └─ 防衛船    （惑星防衛・護衛）
```

**ポイント:**
- 資源は自動では工場に届かない。輸送船が物理的に運ぶ必要がある
- 輸送船が撃墜されると、資源は宇宙空間に散逸（ロスト）
- 工場は資源がなければ停止する
- 占領した工場惑星の生産はそのまま引き継げる

---

## 構造物

### 採掘施設（Mine）

- 建設コスト: 資源 30
- 効果: `mineRate` を +50% 向上させる
- 上限: 惑星ひとつに 2 基まで

### 工場（Factory）

- 建設コスト: 資源 60
- 効果: 船を生産できる。複数工場で生産速度向上
- 複数の工場が同一惑星にあると、生産キューを並列処理できる

### 防衛砲台（Turret）

- 建設コスト: 資源 40
- 効果: 惑星に接近した敵船に自動攻撃（攻撃船・防衛船の代替）
- 占領されると帝国が引き継ぐ

## 船の種類

### パラメータ系統

各船は攻防を4パラメータで表現する。

| パラメータ | 説明 |
|---|---|
| `physAttack` | ミサイル攻撃力 — 相手の `physDef` で軽減される |
| `laserAttack` | レーザー攻撃力 — 相手の `heatDef` で軽減される |
| `physDef` | 物理防御（装甲）— ミサイルダメージを軽減 |
| `heatDef` | 熱防御（シールド）— レーザーダメージを軽減 |

### 輸送船（Transport）

| パラメータ | 値 |
|---|---|
| 積載量 | 50 資源 |
| 速度 | 高速 |
| physAttack / laserAttack | 0 / 0 |
| physDef / heatDef | 低 / 低 |
| 生産コスト | 資源 20 |

役割: 採掘惑星から工場惑星へ資源を運ぶ。護衛なしで移動するとリスクが高い。

### 攻撃艦（Attacker）

| パラメータ | 値 |
|---|---|
| 積載量 | なし |
| 速度 | 中速 |
| physAttack / laserAttack | 高 / 低 |
| physDef / heatDef | 中 / 低 |
| 生産コスト | 資源 35 |

役割: ミサイルによる惑星・工場攻撃および輸送船襲撃。重装甲の防衛艦には弱い。

### 砲艦（Laser Gunship）

| パラメータ | 値 |
|---|---|
| 積載量 | なし |
| 速度 | 低速 |
| physAttack / laserAttack | 低 / 高 |
| physDef / heatDef | 低 / 中 |
| 生産コスト | 資源 40 |

役割: レーザーによる長距離支援・静止目標の破壊。防衛艦が間に立っていない工場・タレットに特に有効。

### 防衛艦（Defender）

| パラメータ | 値 |
|---|---|
| 積載量 | なし |
| 速度 | 低速 |
| physAttack / laserAttack | 低 / 低 |
| physDef / heatDef | 高 / 中 |
| 生産コスト | 資源 28 |

役割: 攻撃艦隊の前に移動して肉盾となる迎撃専門艦。ミサイルを迎撃（ポイントディフェンス）し、惑星手前で敵の火力を吸収する。

## 戦闘システム

### 攻撃の2系統

| 系統 | パラメータ | 特性 |
|---|---|---|
| **物理（ミサイル）** | `physAttack` | 移動体。射程内の移動目標に命中。迎撃可能 |
| **熱（レーザー）** | `laserAttack` | 即着弾。視線上の最前面ターゲットに命中 |

### 射程モデル

| 目標の種別 | ミサイル | レーザー |
|---|---|---|
| **静止目標**（惑星・工場・タレット） | 射程無限 | 射程無限 |
| **移動目標**（船） | 射程あり | 射程あり |

静止しているものは逃げないので距離は問わない。移動目標は距離が離れるほど命中率が下がる。

### ダメージ計算

```
受けるダメージ =
  max(0, physAttack - physDef)
  + max(0, laserAttack - heatDef)
```

### ミサイルエンティティ

ミサイルは視覚的に移動体として描画する。

- 実座標 `(x, y, z)` を持ちターゲットへ向けて移動する
- HPを持ち、**撃ち落とせる**（ポイントディフェンス対象）
- 命中または迎撃で消滅する
- 見た目: 細い光点または短いコーン（Three.js で軽量描画）

### 防衛艦の迎撃配置

防衛艦は常に最良の迎撃ポイントへ動的に移動する。

```
攻撃艦隊 A ──────────────→ 惑星 T
                  ↑
           迎撃ポイント I
           （A→T 経路上、T の手前）
                  ↑
          防衛艦が移動してくる
```

迎撃ポイントの計算:

```
I = T + normalize(A - T) × 迎撃オフセット
```

- 防衛艦が I に位置していれば攻撃を先に受ける
- ミサイルは防衛艦が経路上にいれば吸収される
- レーザーは視線上で最前面にいる艦が先に受ける

### タレットのポイントディフェンス

タレットは惑星固定で2役を担う:

1. **対艦攻撃** — 射程内に接近した敵船を自動攻撃
2. **ミサイル迎撃** — 飛来するミサイルを対艦攻撃より優先して撃墜

複数タレットが揃うほどミサイル攻撃を無効化しやすくなる。

### 惑星占領と施設引き継ぎ

- 守備隊（防衛艦・攻撃艦）を全滅させた時点で占領成立
- 工場・採掘施設・タレットはそのまま占領側が引き継ぐ
- 工場を破壊せずに占領できるため、**稼働中の工場ごと奪うのが最も利益が高い**
- 惑星の資源ストックも占領側のものになる

## 帝国の性格

3帝国でスタート。性格は以下の4種からひとつ。

| 性格 | 特徴 |
|---|---|
| `industrialist` | 工場建設優先。序盤は弱いが後半強い |
| `raider` | 敵の輸送船を狙う。補給線破壊が主戦術 |
| `expansionist` | 惑星占領数を最優先。物流が後回しになりがち |
| `fortifier` | 防衛船・砲台を重視。攻めは遅いが落とされない |

## AIの判断レイヤー

各帝国AIは以下の3レイヤーで判断する:

### 戦略レイヤー（30秒ごと / Ollama）

- どの惑星を次に狙うか
- どこに工場を建てるか
- 輸送ルートをどう設計するか

### 兵站レイヤー（5秒ごと / ルールベース）

- 輸送船が足りているか
- どの工場に資源を優先して送るか
- 護衛を何隻つけるか

### 戦術レイヤー（毎フレーム / ルールベース）

- 近くの敵船を攻撃するか
- 護衛から離れた輸送船を追跡するか
- 劣勢なら撤退するか

## 可視化

### メインビュー（Three.js 3D）

| 要素 | 表示方法 |
|---|---|
| 惑星 | 球体。サイズ = 資源量。色 = 支配帝国 |
| 資源量 | 惑星周囲のリング幅で表現 |
| 工場 | 惑星上の小さな構造物（BoxGeometry） |
| 砲台 | 惑星上の細い棒状構造物 |
| 輸送船 | 小さな四角い船体（BoxGeometry） |
| 攻撃船 | 細長い三角形（ConeGeometry） |
| 防衛船 | 扁平な八面体（OctahedronGeometry） |
| 航路 | 頻繁に使われるルートにうっすら線 |
| 戦闘 | フラッシュエフェクト |

### 観察HUD

- 各帝国: 惑星数 / 資源 / 船数（種別）/ 現在の意図
- 領土比率バー
- イベントログ（補給線切断、占領、工場停止 etc.）

## ランダムイベント

- 彗星通過（航路に一時障害）
- 資源ボーナス（採掘量が一時増加）
- 中立帝国の出現（第4勢力が一時的に活動）
- 工場の故障（修理コストが発生）

---

## スコアと勝利条件

- スコア = 支配惑星数 × 時間
- 全帝国の中で最多惑星を N 秒間維持すると「支配的優位」の表示
- 完全勝利: 全惑星を制圧（idle 観察として終わりは明確にしてよい）
- idle 継続モード: 新しいランダム帝国が参入して続行
