# Escort TD V2 — KING'S ROAD ゲームデザインドキュメント

> ステータス: 設計 / 実装前
> 前版: [GAME_DESIGN_ESCORT_TD.md](GAME_DESIGN_ESCORT_TD.md)(V1 = エスコート型プロトタイプ)
> リサーチ根拠: [REFERENCE_TD_GENRE_RESEARCH.md](REFERENCE_TD_GENRE_RESEARCH.md)

## コンセプト一行

**A都市からB都市へ移動するKINGを、市街地のビルとルートを使って守り抜く放置型エスコートTD。
届けられず倒れても、そこまでのスコアが恒久戦力に変わり、次の護送は必ず強く始まる。**

### 4本柱

1. **TDが主目的** — 移動するKINGの周りで、ビル・ルート・バリケードを使って敵の接近を捻じ曲げる防衛が面白さの核。放置要素はそれを長く遊ばせる器
2. **監視が前進を作る** — 索敵視界と攻撃範囲で進路を十分にカバーできたときだけKINGは進める(美術館問題)。駒ごとに違う攻撃形状で監視の穴を補い合うのが難易度の本体
3. **敗北 = 収穫** — 護送に失敗しても、進んだ距離と撃破数がスコアになり、恒久通貨 CHIP に変換されてユニット枠・新駒・自動化を解放する(The Tower / Rogue Tower 型)
4. **能動的な護衛** — 近衛ユニットがKINGに随伴して敵を「受けて止め」、後衛が焼く。プレイヤーはルート選択・配置・受け位置の指示で介入する(Kingdom Rush 型)

## V1 からの主な変更

V1の「VIPがA→Bへ自動移動する護衛ゲーム」という骨格は**そのまま維持**する。
変えるのは「失敗の意味」と「プレイヤーの介入手段」。

| 項目 | V1(現プロトタイプ) | V2 |
|------|--------------------|-----|
| KING | A→Bへ自動移動 | 同じ(骨格維持)。ただし**分岐点でルート選択**が入る |
| 敗北 | ゲームオーバー(やり直し) | スコア→CHIP変換→恒久強化(メタループ) |
| 操作 | 自動増援のみ | **AIが自動采配(観察ベース)**。プレイヤーは配置/回収/封鎖/ラリー/ルートを任意で上書き |
| 護衛 | 単純な自動随伴 | 近衛リング(前衛/側衛/殿)+ブロック機構+ラリー指示 |
| 敵経路 | 道路沿いに追跡 | ビル・バリケードを障害物として最短経路でKINGへ(迷路の駆け引き) |
| マップ | 21×17 の1画面 | **1区間=1画面固定**を維持し、1都市間=2〜4区間の連結 |
| 進行 | 1ステージ切り | 都市間の路線図キャンペーン(A→B→C…)。区間ごとに難度上昇 |

**流用できる既存実装**: 市街地生成(`buildCity` — メインルート+ループ+分岐の道路網は
ルート選択にそのまま使える)、フローフィールド経路(`bfsFlow`)、チェス駒ユニット定義
(`PIECE`)、敵3種、コマンドモード、Three.js俯瞰シーン一式。

## コアループ(2層構造)

```
【ランループ】= 1回の護送(A都市 → B都市 = 2〜4区間の連結、5〜15分、放置可)
  護送開始
    ├─ KINGが道路網に沿って自動前進(分岐点でルート選択)
    ├─ 区間ゴール到達 → チェックポイント(GOLDボーナス+次区間マップへ乗り継ぎ)
    ├─ 敵が外周・側道からKINGの現在地めがけて最短経路で接近
    │    └─ ビル(破壊不可)とバリケード(プレイヤー設置)が接近路を捻じ曲げる
    ├─ 近衛が接敵ブロック → 後衛・配置ユニットが射撃
    ├─ 撃破でGOLD獲得 → ユニット追加/強化/バリケード建設
    └─ 前進するほど敵の密度・種類が漸増(距離=難易度曲線)
  B都市到達 → クリア(CHIP 1.5倍)/ KING HP 0 → 護送失敗(それでもCHIP獲得)

【メタループ】(護送終了ごと)
  スコア確定 → CHIP変換 → 恒久強化を購入
    ├─ 開始GOLD増 / 初期編成追加(序盤の再走が速くなる)
    ├─ ユニット上限枠+1 / 新しい駒の解放(選択肢が増える)
    └─ 自動化解放(オート増援 → オート配置 → オートリスタート)
  → 同じ区間に再挑戦 or クリア済みなら次の都市区間へ
```

設計意図: メタ強化は「数値を盛る」だけでなく**選択肢と自動化を解放する**方向に置く。
護送を重ねるほどゲームが「操作するもの」から「観察するもの」へ進化し、
シリーズの目的(AI箱庭観察)に収束していく。

## 移動する戦線 — エスコートTD固有の設計

拠点防衛型TDとの最大の違いは**戦線がKINGと共に動く**こと。これを面白さに変換する:

### 陣地転換(リープフロッグ)

- 配置ユニット(Rook/Bishop等の砲台役)は**その場に留まる**。KINGが通過して
  射程外になったユニットは仕事を失う
- プレイヤーは通過済みユニットを**回収**(コストの70%返還)して前方に再配置する。
  「後方を畳んで前方に張り直す」陣地転換のリズムがこのゲーム固有の操作ループになる
- 恒久強化で回収率UP・自動回収を解放 → 放置時はAIが陣地転換を行う(観察の見どころ)

### 前方・側面・後方の3正面

| 正面 | 脅威 | 対応 |
|------|------|------|
| **前方** | 進行先で待ち構える敵集団 | 先行配置(進路の少し先に砲台を張ってから進ませる) |
| **側面** | 側道から湧いてKINGに直行する敵 | バリケードで側道を封鎖し接近路を漏斗化 |
| **後方(殿)** | 追跡してくる大群 | 殿(しんがり)近衛が受け止め、振り切る |

### ルート選択 — 「ルートをどう取るか」

- KINGは分岐点(既存の道路網のループ・分岐)で一時停止し、プレイヤーが次の区間を選ぶ
  (無操作なら数秒後にAIが自動選択 — 放置対応)
- **短いが危険な大通り vs 長いが守りやすい裏道**のトレードオフを地図生成で保証する
  (大通り=スポーン密度高・道幅広く漏斗化しにくい / 裏道=遠回りだがビルの遮蔽が厚い)
- 選択したルートは路面の光る導線で表示。敵の接近予測経路も同時に可視化する

### 前進ゲート — 監視カメラ問題(コアメカニクス)

**「十分に監視できていれば前進できる。監視できていなければ進めない」** —
美術館問題(Art Gallery Problem: 多角形を何台のカメラで見張れるか)の構図を
前進条件そのものにする。これがこのゲームの難易度の本体:

- **霧ルール(V1設計を本採用)**: マップは基本的に霧。**Pawnの索敵視界内にいる敵しか
  他の駒は攻撃できない**。VIP自体は最低限の視界のみ持つ
- **カバレッジ = 索敵視界 × 攻撃範囲の重なり**。駒ごとに攻撃形状
  (扇形/円/四角 — 既存 `AttackShape`)と射程が違うため、
  「どの駒でどの角・どの側道を塞ぐか」の幾何学パズルが配置の本体になる
- **KING(のAI)は進行方向のカバー率がしきい値を超えたときだけ前進する**。
  カバー不足なら停止して増援を待つ。手動の号令で強行前進もできる(視界外から
  一方的に殴られるリスクを取る)
- 湧きは進行連動なので、止まって守るのは安全だが収入も止まる。
  **前進=収入、前進には監視への投資が必要** — この往復がゲームのリズムになる
- HUD に進行方向のカバー率メーターを表示し、未監視エリアは路面を赤く沈ませる。
  「なぜKINGが止まっているのか」が観察で一目で分かること

## KING と近衛システム — 能動的な護衛

### KING

- A都市からB都市へ道路網に沿って自動前進。HP制(恒久強化で上限増)
- 前進の可否は**進行方向のカバー率**で決まる(前進ゲート参照)。
  AIはカバー率が揃うと進み、崩れると止まる — プレイヤーは**「停止/強行前進」の号令**で上書きできる
- 弱い自衛攻撃を持つ(恒久強化ツリーの1系統。「最後は王自ら戦う」絵を作る)
- KING HP 0 で護送失敗 → ただし演出は「収穫画面」(スコア→CHIP変換を主役に)

### 近衛リング(能動防衛の中核)

随伴ユニットはKINGを中心とした**移動フォーメーション**に所属する:

| 役割 | 位置 | 挙動 |
|------|------|------|
| **前衛** | 進行方向前方 | 進路上の敵を先制ブロック。KINGの足を止めさせない |
| **側衛** | 左右 | 側道からの直行敵を受け止める。ラリー指示で受け位置を変更可 |
| **殿** | 後方 | 追跡群を受け止めて時間を稼ぎ、離脱して追いつく(ヒット&アウェイ) |

- **ブロック機構**(Kingdom Rush バラック理論): 地上敵は近衛と接敵すると足が止まる。
  この足止め中が後衛・配置砲台の射撃時間(fire window)。
  「Knightが受け、Rook/Bishopが焼く」分業をシステムとして保証する
- **ラリーポイント**: 側衛・殿の受け位置はKING相対座標でドラッグ指示できる。
  数少ない能動的介入手段として、盤面理解がそのまま強さになる
- 近衛はHPを持ち、倒れるとKINGの隣で再編成(クールダウン)。前線が崩れる→
  KINGを止めて立て直す→再前進、という波が観察の見どころになる

### プレイヤーの介入まとめ(すべて任意 — 観察ベース)

**基本はAIが護送を自動采配する**(ルート選択・増援・配置・陣地転換すべて)。
プレイヤーの介入は全項目が任意で、介入するとAI任せより明確に成績が伸びる:

1. **ルート選択**と**停止/前進の号令**(無操作ならAIが判断)
2. ユニット/バリケードの**配置と回収**(GOLD消費、陣地転換)
3. 側衛・殿の**ラリーポイント指示**
4. **コマンドモード切替**(V1の balanced / ground / air / siege を継承)
5. 護送間の**恒久強化の選択**(ここだけは手動 — メタの意思決定はプレイヤーの領分)

## マップと障害物 — 敵の接近を捻じ曲げる

### 地形レイヤー

- **ビル(破壊不可)**: 既存の市街地生成をそのまま使用。敵は必ず迂回する「天然の壁」
  (They Are Billions の森・山と同じ役割)
- **道路網**: KINGの移動経路(メイン+ループ+分岐 — 既存実装)であり、
  敵の主要接近路でもある。ルート選択の分岐点はこの網の交差点
- **A都市 / B都市**: 出発点と目的地。B都市に近づくほどスポーン密度が上がる

### プレイヤーの配置物

| 配置物 | 役割 | 経路への影響 |
|--------|------|--------------|
| バリケード | 純粋な壁。安価・HPあり | 敵は通行不可(強襲型は攻撃して壊す) |
| 地上ユニット | 戦闘+ブロック | 敵は通行不可(=配置自体が迷路になる) |
| 飛行ユニット | 戦闘のみ | 影響なし(ビル上にも置ける) |

### 障害物のルール(重要な決定)

1. **KINGのルートは塞げない**: KINGの残り進路を完全に塞ぐ配置はUIが拒否する(赤表示)。
   敵の接近路を塞ぐのは自由 — 「敵だけを遠回りさせる」のが配置の腕の見せ所
2. **経路プレビュー必須**: 配置ゴースト表示中、敵フローフィールドの変化を
   光る導線としてリアルタイム表示する。配置の意思決定を「見て分かる」ものにする
3. **漏斗を作らせる**: 側道を1本残して塞ぎ、残した道に火力を集中する
   ファネル戦術(They Are Billions 定石)が自然に成立する道路網を生成時に保証する
4. **障害物への対抗手段を敵に持たせる**(後述: 飛行型・強襲型)。
   完璧な封鎖が唯一解にならないようにする

## ユニット(チェス駒)— 役割の再定義

V1の駒とコストを流用しつつ、「随伴/配置」「ブロック/火力」の分業を明確化する。

| 駒 | 型 | 運用 | 役割 | 特徴 |
|----|----|------|------|------|
| **Pawn** | 飛行 | 随伴 | **索敵(生命線)** | 最安。Pawnの視界内の敵しか他の駒は攻撃できない。数と配置がカバレッジを決める |
| **Knight** | 地上 | 随伴(近衛) | **ブロッカー** | 高HP・近接ガトリング。前衛/側衛/殿を担う |
| **Rook** | 地上 | 配置 | 範囲砲撃 | 迫撃砲。ブロックされた集団を焼く。障害物越し曲射 |
| **Bishop** | 地上 | 配置 | 単体狙撃 | 長射程ミサイル。重装型キラー |
| **Queen** | 飛行 | 随伴(遊撃) | マップ兵器 | 高コスト広域殲滅。1体制限 |

- **ロック&キー回避**(Defender's Quest 原則): どの敵もどの駒でも倒せるが、
  効率が2〜3倍違う、程度のソフトカウンターに留める
- 強化はラン内(GOLD)とメタ(CHIP: 駒レベル・武装分岐解放)の2層

## 敵設計 — 障害に邪魔されながら近づく群れ

V1の「BETA型: 単純ロジック × 物量」思想を継承。全敵の行動は
「フローフィールドに沿ってKING(移動目標)へ直進」だけ。種類の混在で脅威を作る。
**味方=チェス駒 / 敵=無数の群体という非対称は意図的なもの(決定済み)** —
これは大量の敵を薙ぎ倒すゲームであり、敵に個性やモチーフは持たせない。
湧きは**マップ上の固定スポーン口**(道路の外周端・側道の奥)から。
「どこから来るか」が見えることが封鎖・漏斗・監視配置の駆け引きを成立させる。

| タイプ | 経路 | 特性 | 障害物との関係 | 主なカウンター |
|--------|------|------|--------------|----------------|
| **突進型** | 地上・最短経路 | 高速・低耐久・最多数 | 遠回りさせるほど削られる | Rook(範囲)+ Knight(受け) |
| **重装型** | 地上・最短経路 | 低速・高耐久 | 遠回りがDPS時間を稼ぐ | Bishop(単体集中) |
| **強襲型** | 地上・**障害物攻撃** | バリケードを壊して直進する | **封鎖へのカウンター**。壁依存の陣形を罰する | Knightで直接受ける |
| **飛行型** | **障害物無視で直進** | 中速・中耐久 | **封鎖へのカウンター**。対空火力を強制 | Pawn/Queen(飛行組) |
| **要塞型** | 地上・低速 | ボス枠。広範囲攻撃で近衛を薙ぐ | 漏斗で受けると近衛が溶ける | Queen+ラリー分散+先行配置 |

**ウェーブ設計の原則**(リサーチ知見):
- 敵の湧きは**KINGの進行距離に連動**(時間ではなく)。停止して守りを固めるのは
  安全だがGOLD収入も止まる — 前進のリスクとリターンを対にする
- 難所は単種のHPインフレではなく**組み合わせ**で作る
  (例:「突進大群で殿を拘束 + 強襲型が側面バリケードを破壊」)
- 難易度スケーリングの主ノブは**同時攻撃数(スポーン方向数・密度)**。HP倍率は従ノブ
- 区間の中間と終盤に山場(要塞型など)を置き、緊張のリズムを作る

## スコアとメタ進行 — 届けられなくても強くなる

### スコア(護送終了時に確定)

```
score = (通過区間数 + 現区間の進行率) × 区間基準点     ← 主成分。「どこまで届けられたか」
      + 撃破数 × 敵Tier係数
      + 到達時ボーナス(B都市到達で総スコア1.5倍)
CHIP獲得 = floor(score / 100)   ※初回は10分の護送で CHIP 20〜30 程度を想定
```

- どんなに早く倒れても最低数CHIPは入る(**ゼロ収穫の護送を作らない** — 損失回避の無効化)
- 主成分を進行距離にすることで「今回はあの橋まで行けた」という
  前進の実感がそのまま報酬になる

### 恒久強化ツリー(CHIP消費)

インクリメンタル系の定石に従い、**3系統**に分ける:

| 系統 | 内容 | 設計意図 |
|------|------|----------|
| **戦力** | 開始GOLD増 / 初期編成(Knight1体随伴で開始等)/ 駒レベル / KING HP・王の反撃 | 序盤の再走を速くする(プレステージの快感の源) |
| **拡張** | ユニット同時上限+1 / 新駒解放 / 武装分岐解放 / バリケード上限+ / 回収率UP | 選択肢を増やす(Rogue Tower型。数値でなく幅) |
| **AI采配** | 陣地転換AIの解放 / ルート学習(失敗した分岐を避ける)/ ターゲット優先度の最適化 / オートリスタート / 倍速上限解放 | 最初から回るAIを「より賢く・より速く」観察できるようにする |

- 価格は幾何級数(10 → 25 → 60 → …)。「あと1〜2回の護送でアレが買える」が常に見える状態を保つ
- **オート配置AI**は既存の自動増援ロジックの発展形。解放後は「AIの護送采配を
  観察する」遊びになり、シリーズの他ゲーム(colony / planet_strategy)と思想が揃う

### キャンペーン(都市の路線図)

- 世界地図は**都市を結ぶ路線図**(A—B—C—…、途中で分岐あり)。1都市間=1護送ラン
- 1都市間は**2〜4区間(1画面マップ)の連結**。区間の切れ目がチェックポイントで、
  GOLDボーナスと配置の引き継ぎ判断(回収して持ち越すか)が入る
- クリアした都市間の先が解放される。先の都市間ほど区間数が多く・敵構成が凶悪になる
- 未クリア区間はCHIPを貯めて何度でも再挑戦 — 「詰まる→貯まる→越える」が
  キャンペーンの基本リズム(放置ゲーのDDA)

## バランス基準値(初期チューニング表)

Goal Defense の方法論([出典](https://www.gamedeveloper.com/design/balance-in-td-games))を採用。
**HP/ゴールド比を難易度の主変数**とし、以下を初期値とする:

| パラメータ | 初期値 | 根拠 |
|-----------|--------|------|
| 基本敵HP | Pawn 8発分(wave1相当で 30〜40) | 「基本タワーで8発」基準 |
| 撃破報酬 | 敵1体 = 最安ユニット価格の 1/6〜1/8 | 進行1〜2区画ごとに1体追加できる経済 |
| 開始GOLD | 最安ユニット2体+バリケード2枚 | 序盤難易度の最強ノブ。恒久強化の主商品 |
| 湧き成長 | 進行25%ごとに密度+15%、HP+8% | 数の圧力 > HPインフレ |
| スポーン方向 | 序盤: 前方1+側面1 → 進行に応じ最大5方向 | 同時攻撃軸の増加が本体の難易度曲線 |
| ブロック時間 | Knight 1体 = 突進型3体を約4秒拘束 | fire window の基準単位 |
| 回収返還率 | 70%(恒久強化で最大90%) | 陣地転換を「損」に感じさせない下限 |

**チューニング検証式**(湧きが「適切に守れば捌ける」ことの確認):
`(基準弾数 + N) × 射程下タイル数 ≥ h × N`(N=同時敵数、h=1体撃破の所要弾数)

数値は `escort_td_core.ts` の定数群に集約し(現状の構造を維持)、
この表の比率を崩さずに一括スケールできるようにする。

## デザイン・可読性指針(観察体験の最優先事項)

Defender's Quest の原則([出典](https://www.fortressofdoors.com/optimizing-tower-defense-for-focus-and-thinking-defenders-quest/))
+ Emberward の評価点を採用:

1. **1画面固定**: 区間内はスクロールなしの固定俯瞰(現行 21×17 グリッド)を維持。
   都市間の長さは「区間の連結」で表現し、画面外の脅威は絶対に作らない。
   行程全体の現在地はHUDの路線プログレスバーで示す
2. **時間制御**: ポーズ / 1x / 2x / 4x。倍速は放置ゲーの生命線(上限はメタ解放)
3. **視覚アーキタイプ**: 敵は種類ごとに色+シルエットを固定
   (突進=小さく速い赤 / 重装=大きく遅い紫 / 強襲=橙 / 飛行=空色 / 要塞=巨大)。
   駒はチェスのシルエットを強調。「何が起きているか」が引きの画で分かること
4. **完全情報**: ホバーで具体数値ツールチップ(「50%減速」でなく「4秒間、速度半分」)
5. **経路の常時可視化**: KINGの選択ルートと敵の主接近路を路面の光る導線で表示。
   配置・封鎖の結果が絵として見える
6. **KINGの緊張感演出**: KING HP は画面上部の大バー+行程プログレスバー(A→B)。
   無傷=金色 → 危機=赤明滅。遠目の観察でも戦況と進捗が分かる
7. **敗北画面=収穫画面**: リザルトは「GAME OVER」ではなく
   「ここまで届けた距離 → SCORE → CHIP → 何が買えるようになったか」を主役にする

## 放置モードの成立条件

- **最初から介入ゼロで進行する**: オートルート選択+自動増援・自動配置AI
  (既存の自動増援ロジックの発展)が初期状態から護送を回す。放置は解放報酬ではなく前提
- **介入すると明確に伸びる**: 手動のルート選択・封鎖・ラリー指示はAI任せより
  2〜3倍遠くまで届けられる
- **メタ進行が詰まりを解く**: 腕で越えられない区間も、CHIPを貯めれば必ず越えられる
  (放置ゲーのDDA: 詰まり=足踏み=蓄積、であって停止ではない)
- オートリスタート解放後は完全無人周回が可能になり、Electron常駐ウィンドウとして
  「勝手に護送し、勝手に強くなっていく箱庭」が完成する
- **理想像はクッキークリッカー**: 見続けているだけで数字が育ち続けること。
  総撃破数・累計スコア・CHIPの生涯カウンタを常時HUDに置き、メタが育つほど
  1回の護送が長く・遠く・大規模になる(随伴数も敵の物量もインフレしていく)。
  ゲームを閉じている間の進行(オフライン報酬)は**作らない** —
  恒久強化された状態が永続することだけを保証する(localStorage)

## 実装フェーズ

### P0 — メタループ成立(最小の「負けても前進」)
- 護送失敗/成功時のスコア集計(進行距離+撃破)とリザルト画面
- CHIP変換とlocalStorage永続化、最小の恒久強化(開始GOLD / ユニット枠 / KING HP)
- 既存の自動増援AIを維持したまま、手動配置UIを追加
  (グリッドスナップ+配置可否表示+KINGルート封鎖の拒否)

- **霧+索敵カバレッジ+前進ゲート**(Pawn視界内のみ攻撃可、カバー率でKINGが進む/止まる)
- カバー率メーターと未監視エリアの可視化
- 近衛リング(前衛/側衛/殿)とブロック機構

- ユニット回収(陣地転換)と停止/強行前進の号令

- 経路プレビュー(配置ゴースト時のフローフィールド可視化)
- 分岐点でのルート選択UI

### P2 — 放置ゲー化・キャンペーン
- 恒久強化ツリー全実装(3系統・幾何級数価格)

- 区間連結(1都市間=2〜4画面の乗り継ぎ+チェックポイント)
- 都市路線図キャンペーン(都市間の解放・敵テーマ)
- 要塞型ボスと山場設計
- 駒レベル・武装分岐(V1設計の武装システムをメタ解放として実装)

## 決定済み事項(ユーザー確認済み)

- 敵は**固定スポーン口**から湧く(どこから来るかが見える)
- 前進の可否は**監視カバレッジ**で決まる(美術館問題メカニクス)。
  停止への時間ペナルティは設けない
- 霧は**攻撃制約として本採用**(Pawn視界内のみ攻撃可)。難易度の本体は
  駒の攻撃形状・射程を活用して監視の穴を補い合うこと
- 味方=駒 / 敵=群体の**非対称は維持**(大量の敵を倒すゲーム)
- 放置の理想像は**クッキークリッカー**(見続けるほど数字が育つ)。
  オフライン進行は作らないが、恒久強化は永続する
- 実装は**既存 `apps/escort-td/` を直接改修**する
